# Security Rules

适用场景：编辑涉及文件操作、API 调用、用户输入处理的代码时自动加载。

## 当前阶段（Mock）

- **NEVER** — 在 mock 数据中硬编码真实路径或敏感信息
- **IMPORTANT** — 所有 CRUD 操作仅在内存副本上进行，不写入磁盘

## 未来接入真实文件系统时

- **MUST** — 路径遍历检查：用户提供的路径必须做 `realpath` 校验，防止目录穿越攻击
- **MUST** — 文件上传时验证 MIME 类型与扩展名匹配，不允许仅依赖扩展名
- **IMPORTANT** — 用户输入（文件名、搜索关键词）必须转义后注入 DOM，防止 XSS
- **IMPORTANT** — 敏感配置（API Key、数据库连接）通过环境变量注入，禁止硬编码在源码中
- **PREFER** — 大文件操作使用流式处理，避免一次性读入内存
